Ruby Enumerables

March 16, 2015

Enumerables. What the heck does that word even mean? When first reading up on Enumerables, we were supposed to research certain Enumerable methods like map, group_by, and cycle. However, when looking up map in the Ruby docs, I was directed towards the section on Arrays, which confused me even more. Is map a method used on Arrays, or a method under Enumerables? As it turns out, it's really both. An Enumerable is simply a collection of methods which can be used to search or manipulate collections of data (therefore they can be used to search or manipulate Arrays as an example). Therefore to break it down even further, map is a method belonging to a class of methods called Enumerables, which can be used on collections of data such as Arrays and Hashes. Make sense?

There are a few different types of Enumerable methods which do different things:
  • Methods which don't actually touch the underlying data and just go through each item
    • each
  • Methods that pull certain elements out of the underlying data based on criteria and populate a new collection with them
    • select
  • Methods which change all underlying elements through a block and populate a new collection with them. Note this does not alter the original collection.
    • collect
    • map
  • Methods which actually change the original collection of data (are destructive).
    • map!
  • Methods which take a collection of underlying data and reduces it to a single value (i.e. adding up all elements).
    • inject

Getting a little off topic, the original purpose of this post was to discuss one Enumerable, of which I chose map. Map essentially creates a new collection of data (let's call it an Array going forward for ease), using all the original elements of an array, and changes them using a block. The syntax looks like this: 
      array.map {|element| element+2}
What this says is take each element of the original array, add 2 to it, and populate the new value in a new array. So if the original array was [1,2,3,4,5], the new array would look like [3,4,5,6,7]. This is helpful as you can use this method either destructively (using map!), or non-destructively using map.

map can also be used with Hashes, but you must include both the key and value in the code. For example: 
      hash.map {|key, value| value+2}
