JavaScript vs. Ruby

April 8, 2015

This week we got our first glimpse into JavaScript. Objectively speaking, the language is pretty similar to Ruby, and most of the concepts that exist in Ruby also exist in JavaScript, albeit with different syntax. However, as we dug deeper it became more and more clear that JavaScript just doesn't have as many built-in capabilities as Ruby does. I found myself wanting to do something that was so easy in Ruby (like calling a built-in method), and instead having to write the function in JavaScript first and then call it. I remember back to my first week with Ruby and I felt lost because there was so much you could do, it felt infinite. I remember longing for the structure of SQL that I knew, but now that I'm playing in JavaScript, part of me keeps longing to go back and use Ruby instead! I do think that learning Ruby first was extremely helpful though, as catching onto JavaScript has been significantly easier since the underlying principles are very similar.

While similar, the syntax and structure of JavaScript does have some major differences. For example, iterating (looping) through data structures not the same. Whereas Ruby has many built-in methods that can tell how long your data structure is, and go through each instance automatically, JavaScript does not, and you have to tell the code how long you want the loop to go for. Also whereas Ruby has several different ways of going about this (see previous post on Enumerables), JavaScript really has 2 - while statements, and for statements.

Let's go through both examples. First we'll define an array that we want to iterate through, for example: 
      var array = ["1","2","3","4","5"]
A while statement for this array would look like this: 
      var counter = 0;
      while (counter < array.length){
        console.log(array[counter]);
        counter ++;
      }
Going through the code, we first define a variable called counter and set it equal to 0. The text in the parentheses tell the loop how many time to execute. In this case we want the loop to execute the length of the array - 1. So here the length of the array is 5, and the loop will actually run 5 times since we are starting the variable at 0. So it will run when counter is 0, 1, 2, 3, and 4 in order to go through each element of the array (remember the index of the array begins at 0, not 1). The code in the curly brackets then is the code that should be executed. So we are telling the computer, for every loop, print out the element of the array to the console, and add 1 to the counter. If we don't include the last part, the code will just print out the first element of the array to infinity (or until it crashes).

For statements are very similar, but it allows you to write everything on one line instead of breaking it out. So the same example as above but in a for loop would look like this: 
      for (var counter = 0; counter < array.length; counter ++){
        console.log(array[counter]);
      }
Again, to go through the code, the text in the parentheses are just setting the stage for how you want the loop to run, and the text in the curly bracket is the code to be executed each loop. The elements in the parantheses are separated by semi-colons: the first element sets our counter variable to 0, the second element tells the loop to run the length of the array - 1 times, and the last element adds 1 to the counter variable on each run. The code to be executed is just printing out to the console each element of the array.

To illustrate the difference between this and how Ruby executes loops, let's look at the same example in Ruby. We have our array: 
      array = ["1","2","3","4","5"]
Now, instead of having to write all that code like in JavaScript, we can use an easy built-in method like this: 
      array.each {|element| puts element}
With Ruby you don't have to explicitly tell the loop how long to execute for, it will automatically go through every element in the array given. So as mentioned above, while similar, the languages have different capabilities and built-in methods, as well as slightly different syntax.
