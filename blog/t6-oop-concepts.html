Blocs, Procs, and Lambdas

March 31, 2015

Blocs, Procs, and Lambdas - does it sound like we're back in a Frat house in college yet? As opposed to Greek letters, Blocs, Procs, and Lambdas are actually important tools that you can use in Ruby code. They are essentially snippets of code which you tell the computer to do within a method. We've already seen them when using enumerables, but here is an example: 
    def method(arguments)
      arguments.each {|argument| do something to the argument}
    end
The little bit in the curly brackets {} is what we call a Bloc. Blocs function like a method in that they are telling the code to do something, but they are technically undefined and unnamed.

Procs are very similar to Blocs in that they also are bits of code which tell the computer to do something, however they are different in that they are defined and saved as a variable. So you can take the exact same code as in our example above and define it as a variable which then can get passed into the method, instead of writing it out: 
    proc = Proc.new {|argument| do something to the argument}

    def method(arguments)
      arguments.each(&proc)
    end
The only difference here is that you use the "&" sign when you pull the Proc into your method. These can be useful when you want to assign your Proc more than one thing to do, or if you want to refer back to it multiple times in your code.

Last, but not least, we have Lambdas. Lambdas are rarely used, but they are the same thing as Procs, but are more detailed like methods. The other main difference is that you can use the "return" keyword within a Lambda since it will only return the Lambda, not the entire method. If you use "return" in a Bloc or a Proc, it will exit out of the entire method at the return. The other main difference is that Lambdas will check the number of arguments passed into it (like methods), whereas Procs and Blocs do not.
